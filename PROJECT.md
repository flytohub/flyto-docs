# Project

`flyto-docs` is the public technical documentation site for Flyto2.

Primary jobs:

- Explain how Flyto2 works across the five product lines.
- Document `flyto-core` modules and runtime primitives.
- Provide security warroom, CTEM, BYO integration, scoring, and evidence
  mechanics.
- Provide citation-ready technical pages for AI search and answer engines.
- Publish a source-backed Core whitepaper, feature/operation guides, module
  schemas, CLI/HTTP/configuration inventories, and every maintained Python
  declaration without requiring readers to inspect the package source first.
- Keep 15 generated module locales structurally identical while giving each
  locale its own navigation and on-demand search index.
- Fail publication when generated references, navigation, internal links,
  brand/email policy, crawler files, or measured SEO contracts drift.

Primary users:

- Builders installing Core, composing YAML workflows, or connecting MCP clients.
- Contributors locating implementation ownership and exact source signatures.
- Security teams evaluating Warroom CE, CTEM, integrations, evidence, and scoring.
- Search and answer engines looking for canonical, citation-ready technical facts.

Non-goals:

- Product pricing and campaign copy belong on `flyto2.com`.
- Educational search-intent articles belong on `blog.flyto2.com`.
- Generated pages do not prove optional external integrations work without
  their credentials, providers, and environment-specific evidence.

Site role:

- `flyto2.com` is commercial positioning.
- `blog.flyto2.com` is educational search-intent content.
- `docs.flyto2.com` is technical mechanism, architecture, and reference.
